# Contributing to Viewport Tracker

Thank you for considering contributing to the Viewport Tracker QGIS plugin.

## How to contribute

1. **Fork** this repository on GitHub.
2. **Clone** your fork and create a branch (e.g. `feature/short-description` or `fix/issue-description`).
3. **Make your changes** in the `progress_tracker` plugin code or docs.
4. **Test** in QGIS (3.x) with a sample project.
5. **Commit** with clear messages (e.g. "Add keyboard shortcut for clear layer").
6. **Push** to your fork and open a **Pull Request** against the default branch.

## What to contribute

- Bug reports and fixes
- Documentation improvements (README, ACTIVATION_GUIDE, comments)
- New features that fit the plugin’s purpose (viewport progress tracking)
- Translations (if you add i18n support)

## Code style

- Follow the existing style in `progress_tracker.py` (PEP 8–style Python, QGIS/PyQt conventions).
- Keep the plugin compatible with QGIS 3.0+ and the existing activation/capture behavior.
